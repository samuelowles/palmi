# Goal Description
Create a comprehensive business and automation plan to launch and scale a "vibe code" Palm Reading mobile app to $100k MRR. The strategy leverages Rork Max for rapid iOS development, autonomous Luma agents for viral TikTok content creation, and an Hermes agent (running on the Nous Research framework powered by MiniMax 2.7 HighSpeed) for operational automation and UGC management.

## User Review Required
> [!IMPORTANT]

## Proposed PRDs and Documentation

### 1. Rork Max PRD (iOS App Development)
**Objective:** Build a native iOS palm reading app entirely in the browser using Rork Max.
**Core Features:**
- **Onboarding:** Camera integration to capture a photo of the user's palm.
- **Core Loop:** "Vibe code" palm reading generation presented in a sleek, Gen-Z friendly UI.
- **Paywall / Freemium:** 
  - Free: Basic reading (first half).
  - Premium ($2.99 p/m): Unlocks the second half of the reading.
- **Viral Mechanics (The K-Factor):** "Compare with your BF/Best Friend" feature, fully gated behind the $2.99/m subscription. Generates a highly shareable visual compatibility graphic.
- **Documentation Needed for Rork:** 
  - Complete wireframes/UI mockups (optimizing for the "vibe" aesthetic).
  - Technical requirements for device camera access.

### 2. Backend AI & Reading Generation PRD
**Objective:** The core logic engine that converts palm images into highly personalized, accurate-feeling readings.
**Core Features:**
- **Vision Model Integration:** Use **OpenAI Image 2 (GPT Image-2)** as the dedicated palm reading model to directly analyze the user's palm image, interpreting the major lines (heart, head, life) and extracting precise traits.
- **Reading Synthesizer:** Take the extracted visual points and map them to modern astrology/palmistry rulesets using an LLM to output a relatable 2-part reading.
- **Compatibility Engine:** When a user pays $2.99 to "compare with best friend", the engine cross-references the two palm readings to generate a "synergy score" and highly shareable relationship insights.
- **Documentation Needed:** System prompts, deterministic rules for palm line parsing, and JSON schemas for reading outputs.

### 3. Luma Agents PRD (Content Creation Engine)
**Objective:** Automate the daily generation of TikTok slideshows and video assets.
**Core Features:**
- **Story Generation:** Uses an LLM to generate compelling short stories about "how palm reading changed a person's life."
- **Asset Creation:** Prompts **OpenAI Image 2** to create relatable, engaging background visuals for the stories.
- **Formatting:** Assembles the text and images into TikTok-optimized slideshow sequences.
- **Documentation Needed:** Prompt templates for story generation, visual guidelines for OpenAI Image 2 prompting, and strict output formatting (aspect ratio 9:16, min 4 slides).

### 4. Hermes Agent PRD (Operations & Growth Automation)
**Objective:** Autonomous orchestrator running on [Nous Research Hermes Agent](https://hermes-agent.nousresearch.com/) powered by **MiniMax 2.7 HighSpeed**.
**Core Features:**
- **TikTok Distribution:** Posts the slideshows generated by the Luma agents. Manages the "link in bio" dynamically.
- **Engagement Hacking:** Uses secondary "burner" accounts to drop a pinned comment on every viral post: *"palm reading app was the best one they found that does it for free"*.
- **UGC Management:** 
  - Scrapes the optimal UGC platform (determined at launch) to find affordable UGC creators ($10/video).
  - Sends the PRD/brief to the creators automatically.
  - Ingests the submitted UGC, scores them based on engagement potential, and flags the top 10% for paid ad spend.
- **Documentation Needed:** Auth keys for TikTok API, Web automation fallback scripts, UGC platform scraping/API integration specs, and the exact UGC Brief to send to creators.

### 5. Subscription & Payment Automation PRD
**Objective:** Frictionless handling of the $2.99/mo subscription.
**Core Features:**
- **In-App Purchases (IAP):** Integration with RevenueCat or directly with Apple StoreKit to handle the $2.99 subscription without complex manual backend logic.
- **Paywall UI:** High-converting paywall screen presented immediately when the user attempts to view the second half of the reading or compares with a friend.
- **Documentation Needed:** RevenueCat configuration guide and Apple App Store Connect submission checklist.

### 6. Analytics & Financial Flywheel PRD
**Objective:** Track the core metrics (TikTok Views -> App Installs -> Paywall Hits -> Subscriptions) to enable the Hermes agent to allocate UGC ad spend profitably.
**Core Features:**
- **Event Tracking:** Mixpanel or PostHog integration tracking every step of the funnel.
- **Agent Data Feed:** Expose a read-only dashboard/API to the Hermes agent so it can calculate ROI and ROAS on UGC spend and double down on winning creatives.
- **Documentation Needed:** Event taxonomy (e.g., `app_open`, `paywall_view`, `subscription_started`, `friend_share`).

### 7. Automated Support & Community PRD
**Objective:** Zero manual customer support.
**Core Features:**
- **Review Management:** An LLM agent monitors App Store reviews, responding automatically and flagging critical bugs to the developer context window.
- **Email Support:** An AI agent resolving billing issues, refund requests, or app usage questions autonomously.
- **Documentation Needed:** Support knowledge base, tone of voice guide, and refund policy rules.
