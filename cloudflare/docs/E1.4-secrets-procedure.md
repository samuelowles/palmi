# E1.4 — Worker Secrets Setup Procedure

Issue: **#14** (epic 1 → E1.4)
Scope: declare and provision the 4 required Cloudflare Worker secrets plus the
Env interface plumbing. No production secret values are committed to source —
this document only describes the procedure.

---

## 1. Required secrets

| # | Name                       | Purpose                                                      | Source                                  |
|---|----------------------------|--------------------------------------------------------------|-----------------------------------------|
| 1 | `OPENAI_API_KEY`           | OpenAI `gpt-5.4-mini` palm-vision calls                      | platform.openai.com → API keys          |
| 2 | `DEEPSEEK_API_KEY`         | DeepSeek V4 Flash text-synthesis calls                       | platform.deepseek.com → API keys        |
| 3 | `REVENUECAT_WEBHOOK_SECRET`| HMAC for inbound RC webhook signing                          | app.revenuecat.com → Integrations       |
| 4 | `TURNSTILE_SECRET_KEY`     | Cloudflare Turnstile token verification (bot protection)     | dash.cloudflare.com → Turnstile widget |

All four are referenced as `string` bindings on the `Env` interface in
`cloudflare/src/index.ts` (lines 21–24).

> A 5th secret, `JWT_SECRET`, is also referenced in `Env` (line 26) for the auth
> middleware. It is **out of scope for issue #14** and should be tracked under
> its own issue. It is mentioned here only because the running worker will
> fail to start in production if it is unset.

## 2. Why Turnstile is required (not optional)

PRD §5.2 and `docs/AI_RULES.md` §Security both state:

> "Cloudflare Turnstile protects API endpoints from abuse."

The previous `wrangler.toml` comment block and `docs/DEPLOY.md` §1.5 labelled
Turnstile as **optional**. That was incorrect: `/api/read-palm` verifies the
Turnstile token server-side and rejects requests when the secret is missing.
Treat it as required for production.

## 3. Production: `wrangler secret put`

Run these from `cloudflare/`. Each command prompts interactively for the value
(or accept it via stdin from a password manager / CI secret store):

```bash
cd cloudflare
npx wrangler login                                                       # one-time
npx wrangler secret put OPENAI_API_KEY                                   # paste value
npx wrangler secret put DEEPSEEK_API_KEY                                 # paste value
npx wrangler secret put REVENUECAT_WEBHOOK_SECRET                        # paste value
npx wrangler secret put TURNSTILE_SECRET_KEY                             # paste value
npx wrangler secret put JWT_SECRET                                       # paste value (5th, out of #14 scope)

# Verify all 4 #14 secrets are now set:
npx wrangler secret list
```

Expected output of `wrangler secret list`:

```
[
  { "name": "DEEPSEEK_API_KEY" },
  { "name": "JWT_SECRET" },
  { "name": "OPENAI_API_KEY" },
  { "name": "REVENUECAT_WEBHOOK_SECRET" },
  { "name": "TURNSTILE_SECRET_KEY" }
]
```

## 4. Rotation

| Secret                    | Rotation cadence | How                                                 |
|---------------------------|------------------|-----------------------------------------------------|
| `OPENAI_API_KEY`          | 90 days, or on suspected exposure | Regenerate at platform.openai.com → API keys. `wrangler secret put OPENAI_API_KEY` to replace. |
| `DEEPSEEK_API_KEY`        | 90 days, or on suspected exposure | Regenerate at platform.deepseek.com → API keys. `wrangler secret put DEEPSEEK_API_KEY`. |
| `REVENUECAT_WEBHOOK_SECRET` | On suspected exposure only | Rotate inside RevenueCat dashboard; old secret remains valid until re-saved. |
| `TURNSTILE_SECRET_KEY`    | On suspected exposure only | Re-issue widget secret in Cloudflare dashboard. |
| `JWT_SECRET`              | On suspected exposure only | Generate new 32-byte random; `wrangler secret put JWT_SECRET`. All in-flight tokens invalidate. |

## 5. Local development: `.dev.vars`

For `wrangler dev`, secrets are loaded from `cloudflare/.dev.vars` (must be
git-ignored). A template lives at `cloudflare/.dev.vars.example` with
placeholder values.

```bash
cd cloudflare
cp .dev.vars.example .dev.vars
# edit .dev.vars and replace placeholders with real local keys
npx wrangler dev
```

| Concern                  | Production                         | Local dev (`.dev.vars`)              |
|--------------------------|------------------------------------|--------------------------------------|
| Where stored             | Cloudflare encrypted secret store  | `cloudflare/.dev.vars` (gitignored)  |
| Set via                  | `wrangler secret put <NAME>`       | Edit `.dev.vars` directly            |
| Visible to `wrangler list` | Encrypted, name only             | Plain text in file                   |
| Rotation                 | Push new value with same command   | Edit `.dev.vars` and restart dev     |
| Audit trail              | Cloudflare dashboard audit log     | None — local only                    |
